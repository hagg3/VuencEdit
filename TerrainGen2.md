# TerrainGen2 — Implementation Reference

> Source: `~/EdenWorldBuilder/Classes/TerrainGen2.{h,mm}` (~2,917 lines ObjC).
> This document describes the **actual Rust port** in `src-tauri/src/lib.rs`.
> For the original ObjC, the header at `TerrainGen2.h` is the authoritative reference.

---

## Overview

TerrainGen2 is Eden 2.0+'s pre-generated world creator.  The game shipped with
worlds already generated by it (`eden.eden`) rather than running it live.  The
Rust port adds it as a 4th "Tg2" tab in `NewWorldModal.tsx`, giving users access
to the distinctive height-gradient paint style and structures (pyramids, volcanoes,
sky islands) not available in the other generators.

**Key architectural difference from Natural/Classic:** TG2 biome functions *read
back* blocks they've already placed (`makeBeach` floods over `makeGreenHills`,
`makeTransition` blends existing columns).  This requires a full random-access
workspace before any blocks go into the chunk array.

## Data Model (`Tg2Grid`)

```rust
struct Tg2Grid {
    blockz:   Vec<u8>,   // block type at every cell
    colorz:   Vec<u8>,   // paint index at every cell
    gsize:    usize,     // world side in blocks (= size_chunks * 16)
    t_height: usize,     // 64 (legacy) or 256 (New Dawn) — full world height
    vs:       f64,       // vertical scale = t_height/64 (1.0 for 64z)
    amp:      f64,       // user relief multiplier (Tg2Config.amplitude)
    sea_off:  i32,       // additive sea-level offset in blocks (pre-vscale)
}
// Index: x*(gsize*t_height) + z*t_height + y
// Axis:  x=EdenX(east), z=EdenY(south), y=EdenZ(up) — matches ObjC BLOCK(x,z,y) macro
```

**Vertical scaling.** The grid is generated at the **full** world height, not a
fixed 64. Three helpers thread proportional scaling through every biome function
so a 256z world fills the headroom (the same trick the Classic tab uses):
- `g.sv(n)` — scale a vertical block offset / absolute z-band by `vs` (identity at
  vs=1.0, so 64z output is byte-identical to the pre-scaling port).
- `g.relief(a0)` — scale a noise amplitude by `vs * amp`; pass as `a0` to `tg2_fbm2`.
- `g.sea_level(n)` — resolve a water level: `sv(n + sea_off)`, floored at 2.

Fraction-of-`th` baselines (`th/2`, `th/8`, `th/12`, `th/5`) auto-scale because
`th` is the full height; only additive block constants (the `-10`/`-14` offsets,
sea level 19, snow line 34, river/water bands, `fh_cap = th-17`, sky-island float
height 18) are wrapped in `sv`. Inter-zone horizontal tapers in Mix are left
native. Structures (pyramids/volcanoes) keep their native size and self-place on
the scaled surface; the fixed Mix pyramid + sky islands have their base lifted.

After all biome passes, `tg2_flush` copies non-air cells into `WorldGen` chunks:
```rust
// x→wx, z→wy, y→z (TG2 axis → WorldGen axis)
gen.set(x as i32, z as i32, y as i32, bt);
if paint != 0 { gen.set_paint(x as i32, z as i32, y as i32, paint); }
```

**Memory:** 640×640×64×2 bytes ≈ 52 MB for a maximum 40-chunk world — acceptable.
Even extended_z (256z) worlds generate in z=0..63 only; the upper 192 layers stay air.

## Configuration (`Tg2Config`)

```rust
struct Tg2Config {
    seed:          u32,  // seeds ClassicNoise + Rng64; seed2 = seed + 123
    terrain_type:  u8,   // 0–8, see terrain types below
    sky_islands:   bool, // add floating island pass (skipped for Mix — Mix adds its own)
    struct_freq:   u32,  // 0=sparse(0.3×) 1=normal(1×) 2=dense(2×)
    clouds:        bool, // scatter cloud blocks near top of world
    amplitude:     f64,  // relief multiplier, clamped 0.1–4.0 (UI: 0.5–3.0×)
    sea_level_off: i32,  // additive water-level offset, clamped −16..32 blocks
    blend:         bool, // soften Mix/Mtn+River zone seams (bidirectional blur post-pass)
}
```

**Determinism:** same seed + same terrain_type → identical world, because
`ClassicNoise::new(seed)` and `Rng64::new(seed ^ 0xDEAD_C0DE_B16B_00B5)` are
both seeded from `cfg.seed`.

## Paint Cycle Functions

All paint cycles produce a `u8` in `1..=53` (or 0 as a sentinel in cc4/5/6).
They map directly to `PAINT_RGB[1..=54]` — no remapping needed.

### Core formula
```
hue = (idx / div) % 8
if hue >= fold_at { hue = fold_at*2 - 1 - hue }   // fold: 7-hue for fold_at=4
hue += offset
if hue == sentinel { return special_value }
paint = ((hue * 9 + c + 1) % 54) as u8
```

### Per-function table

| Fn | div | fold_at | offset | sentinel | special |
|----|-----|---------|--------|----------|---------|
| `tg2_cc(idx, typ)` | 12 | 4 | +1; c from typ | — | — |
| `tg2_cc2(idx, c)` | 5 | 4 | +1 | — | — |
| `tg2_cc3(idx, c)` | 5 | 4 | +3; if==6→5 | — | — |
| `tg2_cc4(idx, c)` | 5 | 4 | +2 | 6 | return 0 |
| `tg2_cc5(idx, c)` | 5 | 4 | 0 | 6 | return 0 |
| `tg2_cc6(idx, c)` | 1 (no div) | 4 | 0 | 6 | return 0 |
| `tg2_cc7(idx, c)` | 5 | 5 | 0 | — | — |

**Usage by biome:**
- Plains, RiverForest: `cc3` (green/brown gradient)
- Mars: `cc2(h, 0)` (red/orange)
- Mountains: `cc5(h+50, 8)` (greys/purples)
- Beach/Desert: `cc6(h+13, 1)` (sandy yellows)
- Ponies: `cc2(h, 6)` (blues)
- Mix: `cc7(y+10, 8)` for the central stone zone

## Column Fill Helper

```rust
fn tg2_fill_column(g, noise, x, z, h, seed, stone, stone_paint)
```

In the original, `FORMATION_HEIGHT` is set to `h-6` then **immediately overridden**
to `T_HEIGHT-1`.  The condition `y < FORMATION_HEIGHT-16` therefore means `y < 47`
for a 64z world.

**Result:** column is divided into three bands:
- `y < h%2+2` — 3D-noise skin: `n3 < 0.07` → dirt, else air (rough holey surface)
- `h%2+2 ≤ y < T_HEIGHT-17` — solid stone core with `stone_paint`
- `y ≥ T_HEIGHT-17` — 3D-noise skin again (top of formation)

Callers then do a second pass to convert the top-most dirt to grass (block 8).

## Terrain Types (0–8)

### 0 — Plains (`tg2_make_dirt`)
FBM heightmap centred at `T_HEIGHT/2`, freq=2, amp=4, var=3.  Stone body with
`cc2(h, 8)` paint.  Second pass: top dirt cell → grass 8.  Sand bedrock at y=0.

### 1 — Mars (`tg2_make_mars`)
Low flat terrain (`T_HEIGHT/8` offset), same FBM.  Stone body painted `cc2(h, 0)`
(reds/oranges).  Lava (23) floods all air cells at y=0..4.

### 2 — RiverForest (`tg2_make_river_trees`)
Rolling hills offset `T_HEIGHT/2-10`, amp=20.  Dirt body with `cc3` green palette.
Grass top via second pass.  River: any column with air at y=6..15 gets water
(20) at y=1..5.  Trees (`tg2_make_tree2`) placed 1-in-70 on grass surface.

### 3 — Mtn+River
East half (x ≥ gsize/2): `tg2_make_river_trees`.
West half (x < gsize/2 - 32): `tg2_make_mountains`.
Seam (width 32): `tg2_make_transition` smoothstep-blends column heights along a
noise-warped boundary (carries each side's surface paint).

### 4 — Desert (`tg2_make_desert`)
Flat sand at `T_HEIGHT/2-10` (AMPLITUDE=0 in original).  Sand painted `cc6`.
Water (20) floods y=1..16 in all air columns.  Pyramids placed via
`tg2_make_pyramid2` with frequency `(175 × (gsize/300)² × freq_scale).max(1)`.

### 5 — Ponies (`tg2_make_ponies`)
Stone hills with `cc2(h, 6)` blue palette.  Edge tapering: distance from world
edge reduces height.  3D-noise cave carve in lower zone (`tg2_fbm3 > 0` → keep
stone; else → air).  Water (20) fills bottom fifth of world height.

### 6 — Beach (`tg2_make_beach`)
Heightmap biased toward sea level (19).  Sand painted `cc6`.  High-grass columns
(h ≥ 21) get block 8 instead.  Water (20) fills all air at y < 19.  Palm trees
(`tg2_make_palm`) placed 1-in-90 on grass surface above sea level.

### 7 — Mix (`tg2_make_mix`)
Composite layout.  Quadrant calls in order (each overwrites/floods over previous):
1. `tg2_make_green_hills` as base
2. Central stone heightmap (cc7) for interior zones
3. `tg2_make_beach` in bottom-right zone
4. `tg2_make_mars` for right strip
5. Water flood in right-strip lower area
6. Second beach pass
7. `tg2_make_ponies` bottom-left
8. `tg2_make_classic_gen` left interior
9. `tg2_make_desert` left-center strip (no pyramids — freq=0)
10. `tg2_make_mountains` top-left corner
11. Pyramid scatter in left interior zone
12. Fixed pyramid at `(gs/4, 3gs/4, h=25, sy=17)`
13. Trees in classic-gen area (1-in-50 on grass)
14. Sky islands in top-center band (40 islands)
15. Volcanoes in right area
16. Bedrock floor (y=0 → block 1) across entire world
17. Global sparse tree pass (1-in-300 on grass)

### 8 — Flat
Bedrock at y=0 and y=1 only (from the `clear()` pass at the start of every world).

## Structure Functions

### `tg2_make_pyramid(g, cx, cz, h)`
Square-footprint stepped pyramid (slate/14).  Scans downward from top looking for
a sand (4) base; aborts if it finds non-sand, non-air.  Shrinks radius by 1 each
layer up.

### `tg2_make_pyramid2(g, cx, cz, h, _color, sy)`
Diamond footprint: `(sx-cx).abs() + (sz-cz).abs() <= r`.  When `sy == -1`
searches downward for a sand base the same way as `tg2_make_pyramid`.  `_color`
parameter is accepted but ignored (always slate).

### `tg2_make_volcano(g, cx, cz, base_y, start_radius, rng)`
Cone built from `start_radius` down to 1.  At each radius:
- Edge check: `((i²+j²) as f64) < (r2 + 3*sin(12*atan2(i,j)))²` → stone (2) with paint 36
- Interior: `i²+j² < r2²` → lava (23)

Then a cap of lava is placed above the peak using decreasing radii with noise jitter.

### `tg2_make_sky_island(g, cx, cz, r, rng)`
Lower hemisphere of a sphere (`x²+z²+y²<=r²`, y from `-r` to `-r/2`).
Top ring (`y == -r/2`): grass (8) surface, palm tree placed at 1-in-90.
Interior: sand (4) with `cc3` paint.
Centre y of sphere: `cy = 18 + r - r/4 - r/8` (lifts island to visible sky height).

### `tg2_make_palm(g, x, z, y, hh, rng)`
Trunk: coloured wood (7) with paint from `[2,0,29,38]`.  Height `hh + rng%4`.
Crown: 5-arm star pattern at trunk top, one block high, leaves (5) with paint from
`[0,31,22,40]`.  Each arm extends to distance 3.

### `tg2_make_tree(g, x, z, y, rng)` / `tg2_make_tree2(g, x, z, y, hh, rng)`
Standard deciduous tree.  Trunk (6) for bottom ¾ of height.  5×5×(upper ¼) leaf
box, corners trimmed and outer-edge 50% sparse.
- `tg2_make_tree`: height 6–9, leaf palette `[0, 19, 20, 21]`
- `tg2_make_tree2`: height `hh + rng%4`, leaf palette `[0, 31, 40, 40]` (pine greens)

## Green Hills Sub-function

```rust
fn tg2_make_green_hills(g, noise, seed2, height)
```

Used only by `tg2_make_mix` as the base pass.  Fills x: `gs/4-15..gs` and
z: `0..3gs/4+15` with dirt columns (cc3 green), with edge tapering logic to
smoothly lower terrain near the world border.  Then converts top dirt to grass.
Then fills a water lake in the left-centre zone (x: `gs/4..gs/2-60`, y: 6..18).

`seed2 = seed + 123` is passed here so the hills noise is independent of the
seed noise used by the overlaid biomes.

## Generate Flow (`generate_tg2_world`)

```
1. gsize = size_chunks * 16
2. t_height = min(world t_height, 64)   // always generate in lower 64z
3. noise = ClassicNoise::new(seed)
4. rng   = Rng64::new(seed ^ 0xDEAD_C0DE_B16B_00B5)
5. g = Tg2Grid::new(gsize, 64)
6. Bedrock floor: y=0,1 → block 1 across entire grid
7. Scale structure counts: pf = (175 * (gsize/300)² * ff).max(1)
                            vf = (20  * (gsize/300)² * ff).max(0)
8. Dispatch on terrain_type → biome function(s)
9. If sky_islands && terrain_type != 7: scatter tg2_make_sky_island passes
10. If clouds: tg2_place_clouds (cloud blocks at ~4/5 of world height)
11. Enforce bedrock floor again (biome functions may overwrite y=0)
12. WorldGen chunks allocated; tg2_flush(g, gen)
13. surface_z = scan column (gsize/2, gsize/2) downward for first non-air cell + 1
14. Return surface_z (used as player spawn height by write_world_file)
```

## Cloud Placement (`tg2_place_clouds`)

Scatters rectangular cloud (19) patches near `T_HEIGHT * 4/5`.  Patch count ≈
`(gsize² / 500).max(2)`.  Each patch is 6–18 blocks wide/deep at a random position.
The slab altitude is jittered ±2 blocks per patch (clamped to `[th/2, th-2]`) so
clouds don't all sit on one flat plane, and any cell where terrain already rises
into the cloud layer is skipped (`g.get(px,pz,cy)!=0` → continue) so a cloud never
buries a mountain top.

---

## Transition Seam (`tg2_make_transition`)

Used by terrain type 3 (Mtn+River) to blend the height boundary between mountains
and river-forest.  Signature: `tg2_make_transition(g, noise, seed, sx, sz, ex, ez)`.
For each z row it reads the surface `(height, block, paint)` of the left reference
column (`x = sx-1`) and right reference column (`x = ex`), then for each x in the
band:

- **Noise-warped seam** — the normalised seam position is offset by
  `tg2_fbm2(noise, x, z, seed+533, 1.0, span*0.25, 1.0)`, so the boundary wobbles
  across rows instead of tracing a straight column (covered by the
  `tg2_warped_borders_not_axis_aligned` test).
- **Smoothstep height ramp** — `s = fx²(3-2fx)` eases the height interpolation in/out
  rather than ramping linearly.
- **Paint carried across** — the chosen side contributes both its block type *and*
  its surface paint (`s < 0.5` → left, else right), so the seam keeps the natural
  colour of whichever biome it resolves to.

## Deviations from Original

| Aspect | Original ObjC | Rust port |
|--------|--------------|-----------|
| World size | Fixed `GSIZE = T_SIZE * 10 ≈ 300` | User-selectable 5–40 chunks (80–640 blocks) |
| Structure frequency | Hard-coded per-biome counts | Scaled by `(gsize/300)²` × `struct_freq` |
| `makeTransition` colour | `lookupColor(blended RGB)` | Smoothstep + noise-warped seam; carries each side's surface paint |
| `makeWorm` | Physics-simulated worm tunnels | Not ported (complex, visual impact minimal) |
| `genTemperatureMap` | 16×16 Perlin → bilinear upsample | Not needed — TM() checks are dead code |
| `makeCave` (standalone) | Local cavern carve | Not exposed; Ponies uses inline 3D noise instead |
| `g_terrain_type` default | Hardcoded 7 in `tg2_init()` | UI default 7 (Mix), user-selectable |
| Seed | `DEFAULT_LEVEL_SEED = 333333` | User-selectable, randomise button |
| `makeVolcano` loop | `for(int radius=start_radius; radius>0; radius--)` | `(1..=start_radius).rev()` |
| Rng | ObjC `arc4random()` | `Rng64` xorshift64 (deterministic per seed) |

## Adding a New Terrain Type

1. Write `fn tg2_make_myterrain(g: &mut Tg2Grid, noise: &ClassicNoise, seed: f64, ...)`.
2. Add arm to the `match cfg.terrain_type { ... }` block in `generate_tg2_world`.
3. Add a preview arm to `preview_tg2_world` (heightmap + surface block type only).
4. Add button to the 3×3 grid in `NewWorldModal.tsx` (currently hardcoded by index).

If the new type uses structures, wire up the `pf`/`vf` scale variables or add a
separate count that follows the same `(gsize/300)² × freq_scale` pattern.

## Known Issues / Future Work

- **Mix layout is approximate.** The original `makeMix()` in ObjC has exact pixel-level
  coordinate math for each zone boundary; the port reconstructs these from reading the
  source but some zone edges may differ by a few blocks.
- **Biome *material* borders stay axis-aligned.** Block-type boundaries between Mix
  zones (e.g. desert sand meeting grass) remain straight lines — only the *heights*
  (via `tg2_blend_seams`) and the explicit Mtn+River seam (via `tg2_make_transition`)
  are noise-warped.  A full domain-warped biome-ID map was evaluated and deliberately
  **not** implemented: `tg2_make_mix` composes zones by reading back already-placed
  blocks in a fixed order, so warping the zone footprints would require a generator
  rewrite, against the project's "pragmatic edge-blending only" guidance.  The
  height blur + paint dither already remove the most jarring cliff/colour artifacts.
- **No `makeOcean`** — the original has a `makeOcean()` function (referenced in header)
  but it was not found in the ~2917 lines read.  Beach's water fill covers this use case.
- **Extended_z worlds now scale** to fill the full 256z (via `vs = t_height/64`).  This
  is a deviation from the original (T_HEIGHT=64 only).  Mix's inter-zone horizontal
  tapers stay native, so a few zone edges in 256z Mix may ramp slightly differently
  than a perfectly-scaled version would.
- **Preview parity.** `preview_tg2_world(size_chunks, seed, terrain_type, max_px,
  custom_biomes, extended_z, amplitude, sea_level_off)` now reflects the height
  format (`th`/`vs`), amplitude (relief multiplier) and sea-level offset, so the
  preview tracks those sliders.  It remains a fast heightmap-only approximation:
  **no** fill, caves, structures, trees or `blend` are shown.
- **Biome blend (`blend`)** is a bidirectional smoothing post-pass
  (`tg2_blend_seams(g, noise, seed, iters)`, ~`6 × vs` iterations).  Each iteration
  snapshots every natural-terrain (`2|3|4|8`) column's surface, then for each column
  computes a noise-warped, box-blurred (`radius = clamp(round(vs*2), 2, 5)`) target
  height and moves halfway toward it — **raising** low columns (stacking the column's
  own material) *or* **carving** high columns (to air).  Surface paint is re-tinted
  with a hash-dithered pick from the kernel's neighbours, so a seam between two
  palettes resolves to a speckled gradient.  Verified by
  `tg2_blend_smooths_both_directions` (mean neighbour-height delta drops after blend).
